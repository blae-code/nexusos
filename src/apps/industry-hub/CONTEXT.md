# Industry Hub — App Context

**Route:** `/app/industry` (with `?tab=overview|materials|blueprints|craft|refinery`)
**Directory:** `src/apps/industry-hub/`
**Status:** COMPLETE — production-ready active path only

---

## Purpose

NexusOS industrial operations dashboard for Redscar Nomads. Covers:
- Org material stockpile management (INVENTORY, REFINERY, OCR ingestion)
- Blueprint registry with gap analysis, recipe cross-reference, and craft queueing
- Searchable crafting reference guide with blueprint prototyping against live stock or user-defined quality assumptions
- Refinery batch tracking with live countdown timers
- Craft queue management with AI-powered optimisation
- Patch digest display (RSI Comm-Link → Claude API → structured changes)
- System readiness insight strip (generated by `generateInsight` Base44 function)

## User Flows

1. **View overview** — Landing tab; shows: stat cards (stockpile SCU, avg quality, blueprint coverage), insight strip (generateInsight), top 5 materials, priority blueprints, scout intel rows
2. **Browse stockpile** — MATERIALS tab → sortable/filterable table; search by name; filter by type (RAW/REFINED/SALVAGE/CRAFTED), quality slider, status (T2-ELIGIBLE / BELOW T2 / REFINE FIRST)
3. **OCR screenshot** — MATERIALS tab → drag-drop screenshot → ocrExtract → if INVENTORY: review table → confirm → bulk Material.create; if MINING_SCAN: redirected to Scout Intel
4. **Add blueprint** — BLUEPRINTS tab → Pioneer+ → AddBlueprintDialog → item autocomplete → category/tier/holder → recipe ingredients → Blueprint.create
5. **Set priority** — Pioneer+ clicks star on BlueprintRow → Blueprint.update(is_priority) → surfaces in PriorityPanel and GapSidebar
6. **View recipe** — Click blueprint row → RecipePanel expands → shows each ingredient with IN STOCK / LOW QUAL / NO STOCK status cross-referenced against Materials
7. **Queue craft job** — RecipePanel "CRAFT THIS" → CraftQueue.create; or manually from CraftQueueTab
8. **Claim craft job** — CRAFT QUEUE tab → CLAIM button → CraftQueue.update(status=CLAIMED, claimed_by_callsign)
9. **Optimise queue** — CRAFT QUEUE tab → OPTIMISE → craftingOptimiser function → reorders by priority + material feasibility
10. **Submit refinery batch** — REFINERY tab → select raw material, qty, method, station → forecast shown → submit → RefineryOrder.create + output Material.create
11. **Collect refinery batch** — REFINERY tab → order status READY → COLLECT button → RefineryOrder.update(status=COLLECTED)
12. **Read patch digest** — PatchDigestHeader auto-loads latest PatchDigest; shows severity, industry summary, affected systems, key changes

## What NOT to Touch

- **Do not** change the OCR extraction flow in `Materials.jsx` — it relies on `ocrExtract` returning specific `screen_type` values (`INVENTORY`, `REFINERY_ORDER`, `MINING_SCAN`, `CRAFT_QUEUE`, `SHIP_STATUS`). These match the Base44 function schema exactly.
- **Do not** move `RecipePanel.jsx` — it exports `ingredientStatus()` which is imported by `PriorityPanel.jsx` and `GapSidebar.jsx` via relative paths.
- **Do not** change T2 eligibility threshold — it is `quality_pct >= 80` hardcoded in `Materials.jsx` and `IndustryOverviewWidgets.jsx`. This is a hard game mechanic from CLAUDE.md.
- **Do not** label the InsightStrip as "AI" — it renders as "OP READINESS" or similar. The Claude API call is hidden; see CLAUDE.md AI visibility rule.
- **Do not** use the `generateInsight` org-readiness mode outside the InsightStrip context — Scout Intel now uses the separate `context: 'scout_route'` branch for tactical deposit guidance.
