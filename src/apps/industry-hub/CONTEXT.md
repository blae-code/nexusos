**Route:** `/app/industry` (with `?tab=overview|materials|blueprints|craft|refinery`)
**Directory:** `src/apps/industry-hub/`
**Status:** COMPLETE ‚Äî production-ready; two in-progress alternative views (see below)

---

## Purpose

NexusOS industrial operations dashboard for Redscar Nomads. Covers:
- Org material stockpile management (INVENTORY, REFINERY, OCR ingestion)
- Blueprint registry with gap analysis, recipe cross-reference, and craft queueing
- Refinery batch tracking with live countdown timers
- Craft queue management with AI-powered optimisation
- Patch digest display (RSI Comm-Link ‚Üí Claude API ‚Üí structured changes)
- System readiness insight strip (generated by `generateInsight` Base44 function)

## Entity Ownership

This app is the **primary owner** of:

| Entity | Operations |
|--------|-----------|
| `Material` | create (bulk OCR / manual), read, update (edit, archive), delete |

## User Flows

1. **View overview** ‚Äî Landing tab; shows: stat cards (stockpile SCU, avg quality, blueprint coverage), insight strip (generateInsight), top 5 materials, priority blueprints, scout intel rows
2. **Browse stockpile** ‚Äî MATERIALS tab ‚Üí sortable/filterable table; search by name; filter by type (RAW/REFINED/SALVAGE/CRAFTED), quality slider, status (T2-ELIGIBLE / BELOW T2 / REFINE FIRST)
3. **OCR screenshot** ‚Äî MATERIALS tab ‚Üí drag-drop screenshot ‚Üí ocrExtract ‚Üí if INVENTORY: review table ‚Üí confirm ‚Üí bulk Material.create; if MINING_SCAN: redirected to Scout Intel
4. **Add blueprint** ‚Äî BLUEPRINTS tab ‚Üí Pioneer+ ‚Üí AddBlueprintDialog ‚Üí item autocomplete ‚Üí category/tier/holder ‚Üí recipe ingredients ‚Üí Blueprint.create
5. **Set priority** ‚Äî Pioneer+ clicks star on BlueprintRow ‚Üí Blueprint.update(is_priority) ‚Üí surfaces in PriorityPanel and GapSidebar
6. **View recipe** ‚Äî Click blueprint row ‚Üí RecipePanel expands ‚Üí shows each ingredient with IN STOCK / LOW QUAL / NO STOCK status cross-referenced against Materials
7. **Queue craft job** ‚Äî RecipePanel "CRAFT THIS" ‚Üí CraftQueue.create; or manually from CraftQueueTab
8. **Claim craft job** ‚Äî CRAFT QUEUE tab ‚Üí CLAIM button ‚Üí CraftQueue.update(status=CLAIMED, claimed_by_callsign)
9. **Optimise queue** ‚Äî CRAFT QUEUE tab ‚Üí OPTIMISE ‚Üí craftingOptimiser function ‚Üí reorders by priority + material feasibility
10. **Submit refinery batch** ‚Äî REFINERY tab ‚Üí select raw material, qty, method, station ‚Üí forecast shown ‚Üí submit ‚Üí RefineryOrder.create + output Material.create
11. **Collect refinery batch** ‚Äî REFINERY tab ‚Üí order status READY ‚Üí COLLECT button ‚Üí RefineryOrder.update(status=COLLECTED)
12. **Read patch digest** ‚Äî PatchDigestHeader auto-loads latest PatchDigest; shows severity, industry summary, affected systems, key changes

## What NOT to Touch

- **Do not** change the OCR extraction flow in `Materials.jsx` ‚Äî it relies on `ocrExtract` returning specific `screen_type` values (`INVENTORY`, `REFINERY_ORDER`, `MINING_SCAN`, `CRAFT_QUEUE`, `SHIP_STATUS`). These match the Base44 function schema exactly.
- **Do not** move `RecipePanel.jsx` ‚Äî it exports `ingredientStatus()` which is imported by `PriorityPanel.jsx` and `GapSidebar.jsx` via relative paths.
- **Do not** change T2 eligibility threshold ‚Äî it is `quality_pct >= 80` hardcoded in `Materials.jsx` and `IndustryOverviewWidgets.jsx`. This is a hard game mechanic from CLAUDE.md.
- **Do not** label the InsightStrip as "AI" ‚Äî it renders as "OP READINESS" or similar. The Claude API call is hidden; see CLAUDE.md AI visibility rule.
- **Do not** use the `generateInsight` function outside the InsightStrip context ‚Äî it reads full org state and is rate-sensitive.
