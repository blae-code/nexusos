# NexusOS

NexusOS is the Redscar Nomads operations console for Industry, Scout Intel, Op Board, Rescue, Armory, Commerce, and Logistics workflows. The app is Base44-backed and now uses invitation-based access with issued usernames and auth keys.

The UI direction targets an in-verse Star Citizen operations console: ambient shell depth, restrained color, MFD-style panels, and animated live data rather than generic web-app chrome.

## Local Development
1. Clone the repository.
2. Run `npm install`.
3. Create `.env.local` with the frontend Base44 variables:

```env
VITE_BASE44_APP_ID=your_app_id
VITE_BASE44_APP_BASE_URL=your_backend_url
```

4. Start the client with `npm run dev`.
5. Sign in through the Access Gate with a real issued username and auth key.

For initial environment recovery, bootstrap or repair the fixed `system-admin` account through `/setup` or from `/app/keys` once an admin session exists.

## Auth Model
- Users are invited by a Pioneer/admin user.
- Each invite issues:
  - a fixed username used for login
  - an initial callsign used as the display identity
  - a rank
  - a non-expiring auth key
- Callsigns are mutable in-app.
- Auth keys are hashed, non-retrievable, and can only be revoked or regenerated by Pioneer/admin users.
- Onboarding runs once per user and is skipped on later sign-ins after completion.

## Release and Verification
- `npm run typecheck`
- `npm run lint`
- `npm run build`
- `npm run base44:check`
- `npm run version:check`

Use `.\version-bump.ps1 patch|minor|major "release note"` for releases. Version metadata is kept in sync automatically from `version.json`.

## Deployment
Publish from Base44 after the branch is green and the required auth/runtime environment is configured.

GitHub remains the source of truth. Any Base44 editor polish must be mirrored back into the repo immediately after publishing.

Base44 docs: [https://docs.base44.com/Integrations/Using-GitHub](https://docs.base44.com/Integrations/Using-GitHub)
